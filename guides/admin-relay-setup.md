# Развёртывание relay-сервера

Relay — сигнальный сервер. Он помогает двум узлам найти друг друга при первом подключении. Через него трафик **не проходит** — только координация. После установки соединения узлы общаются напрямую.

---

## Требования

- Linux-сервер с публичным IP
- Docker 24+ и Docker Compose v2
- Открытый входящий порт TCP **7778** (или любой другой на ваш выбор)

---

## Шаг 1 — Клонируйте репозиторий

```bash
git clone https://github.com/niklod/kin.git
cd kin
```

---

## Шаг 2 — Запустите relay

```bash
docker compose up -d
```

Docker соберёт образ и запустит контейнер. При первом запуске relay автоматически генерирует Ed25519-идентификацию и сохраняет её в Docker volume `relay-data`. При последующих запусках NodeID не меняется.

Проверьте, что всё работает:

```bash
docker compose logs relay
```

Вы должны увидеть:

```
relay running
  NodeID: a3f9c1...
  Listen: 0.0.0.0:7778
```

---

## Шаг 3 — Убедитесь, что порт доступен

```bash
# С другой машины:
nc -zv <IP_сервера> 7778
```

---

## Параметры запуска

По умолчанию relay слушает `0.0.0.0:7778`. Чтобы изменить адрес или порт, раскомментируйте строку `command` в `docker-compose.yml`:

```yaml
command: ["--listen", "0.0.0.0:9000", "--config-dir", "/data"]
```

| Флаг | По умолчанию | Описание |
|---|---|---|
| `--listen` | `0.0.0.0:7778` | Адрес и порт для прослушивания |
| `--config-dir` | `/data` (в контейнере) | Каталог для хранения идентификации |

---

## Адрес для пользователей

Сообщите пользователям адрес relay в формате `host:port`:

```
<публичный_IP>:7778
```

или доменное имя, если есть:

```
relay.example.com:7778
```

---

## Управление контейнером

```bash
# Остановить
docker compose stop

# Остановить и удалить контейнер (данные сохранятся в volume)
docker compose down

# Посмотреть логи
docker compose logs -f relay
```

---

## Обновление

```bash
git pull
docker compose build --no-cache
docker compose up -d
```

Идентификация (NodeID) при этом **сохраняется** — volume `relay-data` не затрагивается.
