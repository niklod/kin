# Начало работы с Kin

Kin — это программа для обмена файлами напрямую между двумя компьютерами. Никаких облаков, никаких сторонних сервисов. Файлы идут прямо от вас к другому человеку.

---

## Что вам понадобится

- Бинарник `kin` (попросите у администратора или соберите сами: `go build -o kin ./cmd/kin`)
- Адрес relay-сервера от администратора, например: `relay.example.com:7778`

---

## Шаг 1 — Запустите Kin

Откройте терминал и запустите:

```bash
./kin --relay relay.example.com:7778 run
```

Вы увидите примерно следующее:

```
kin running
  NodeID: 7f3a9c...
  Listen: 0.0.0.0:7777
  Shared: /home/user/Kin
  Relay:  relay.example.com:7778
```

**Kin запущен.** Оставьте это окно открытым — программа работает в фоне.

> Папка `~/Kin` создаётся автоматически. Всё, что вы туда положите, будет доступно для передачи другим.

---

## Шаг 2 — Положите файлы в общую папку

Скопируйте нужные файлы в папку `~/Kin` (или `C:\Users\имя\Kin` на Windows).

```bash
cp important-document.pdf ~/Kin/
```

---

## Шаг 3 — Пригласите другого пользователя

Чтобы кто-то мог подключиться к вам, сгенерируйте одноразовую ссылку-приглашение:

```bash
./kin --relay relay.example.com:7778 invite
```

Вы получите строку вида:

```
kin:AAAB3NzaC1yc2EAAAADAQABAAABgQ...
```

**Отправьте эту строку другому пользователю** — через мессенджер, почту, любым удобным способом.

> Ссылка действует **24 часа** и может быть использована только один раз.

---

## Шаг 4 — Подключитесь к другому пользователю

Если вам прислали ссылку-приглашение, подключитесь к отправителю:

```bash
./kin --relay relay.example.com:7778 join kin:AAAB3NzaC1yc2EAAAADAQABAAABgQ...
```

Вставьте строку целиком вместо примера выше.

Если всё прошло успешно:

```
connecting to peer 7f3a9c...
connected to 7f3a9c
```

Готово — теперь вы связаны с этим пользователем.

---

## Шаг 5 — Проверьте статус

Чтобы посмотреть, с кем вы сейчас связаны:

```bash
./kin status
```

Пример вывода:

```
NodeID: 7f3a9c...
Peers:  1
  a3f9c1  trusted  2026-04-04 12:34:56
```

---

## Частые вопросы

**Нужно ли держать программу запущенной?**
Да. Пока `kin run` работает, вы доступны для других пользователей и можете получать файлы.

**Что если оба за NAT (домашний роутер)?**
Relay-сервер помогает установить прямое соединение даже за NAT. Если соединение не удаётся — обратитесь к администратору.

**Как остановить Kin?**
Нажмите `Ctrl+C` в окне, где запущен `kin run`.

**Куда сохраняются настройки?**
- macOS / Linux: `~/.config/kin/`
- Windows: `%APPDATA%\kin\`

Там хранится ваш уникальный идентификатор (NodeID). Не удаляйте эту папку.

---

## Краткая шпаргалка

```bash
# Запустить
./kin --relay relay.example.com:7778 run

# Сгенерировать приглашение
./kin --relay relay.example.com:7778 invite

# Принять приглашение
./kin join kin:TOKEN...

# Посмотреть статус
./kin status
```
